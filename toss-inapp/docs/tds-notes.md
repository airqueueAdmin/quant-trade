# TDS Notes

토스 인앱 프론트 작업 원칙:

- UI 구현은 `TDS(Toss Design System)`를 기준으로 진행합니다.
- 임시 커스텀 스타일을 먼저 만들고 나중에 교체하는 방식은 지양합니다.
- 페이지 레이아웃보다 사용자 흐름을 먼저 설계하고, 그 위에 TDS 컴포넌트를 배치합니다.
- 기존 `frontend/` Streamlit UI는 참고만 하고 그대로 옮기지 않습니다.

초기 설계 체크포인트:

- 상단 헤더 구조
- 탭 또는 세그먼트 전환 구조
- 입력 폼 패턴
- 리스트/카드 패턴
- 하단 고정 CTA 필요 여부
- 로딩/에러/빈 상태 표현 방식

후속 작업에서 정할 것:

- 실제 프론트엔드 프레임워크
- TDS 패키지/적용 방식
- 공용 테마 또는 토큰 관리 방식
